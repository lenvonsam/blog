工程项目结构

  • 供应链工程后端项目基于 springcloud 微服务架构搭建，依托于 SVN 进行代码管理，下面就整个工程项目模块做个解释说明。

  • 单个微服务模块标准化项目结构说明(拿采购模块模块做示例说明)，单个项目技术架构 springboot+mybatis+springcloud+oracle

开发环境准备

  • 有较强的英文阅读能力！！！
  • 有一定自我解决问题的能力！！！
  • jdk8 [官方下载] https://www.oracle.com/technetwork/java/javase/downloads/jdk8-downloads-2133151.html ，并配置到环境变量中(自行搜索教程)
  • maven3.3 [官方下载] http://maven.apache.org/download.cgi，并配置到环境变量中(自行搜索教程)
  • intellij ce [官方下载] https://www.jetbrains.com/idea/ (下载 community 版就足够了)，并配置 svn(自行搜索教程)
  • java 开发规范 [在线文档] https://blog.thinkingsam.cn/2019/04/09/java-rule/
  • springcloud 简单教程 [在线文档] https://blog.thinkingsam.cn/tags/SpringCloud/
  • mysqlplus 简单教程 [在线文档] https://mp.baomidou.com/
  • springboot 简单教程 [在线文档] https://docs.spring.io/spring-boot/docs/2.1.11.RELEASE/reference/htmlsingle/
  • api restful 规范 [在线文档] https://blog.thinkingsam.cn/2019/04/09/api-rule/

开发规范与要求说明

1、 所有开发人员必须使用Intellij IDE进行本地开发，并要求安装Alibaba Java Coding Guidelines插件(自行搜索教程)

2、所有文件的命名规范请参照 java 开发规范，所有文件命名风格请参照示例工程文件命名风格

3、所有工程目录结构说明中标注不做更改的，请不要随意做修改！！！

4、开发人员注意单个实体类(比如 user 类)都可以使用 GeneratorFile 文件自动生成 entity/mapper/service/controller无需自己手动创建，希望开发人员更多关注业务模块的开发

5、所有 Controller 进行数据流通信的是时候涉及到类对象都是核心文件库里面定义的 Dto 类对象，切勿将 entity 里面数据库对象对外暴露！！！

6、在第一次接触新项目微服务模块时，最好与开发过的小伙伴进行沟通交流，尤其是规范的注意事项

7、多人协作开发时，开发人员在开发业务模块前，若之前有不同的开发人员进行开发，需与其进行沟通交流，确保二次开发原有功能模块都能正常运行

8、每个开发人员在开发完功能任务模块，需自测(自测不仅仅是正常流程，异常流程尤其重要)确保其功能模块正常运行才能提交给测试人员进行测试
